# 기능 로드맵

구현 예정 기능 목록. 우선순위 순으로 정렬.

---

## 🔴 P1 — GitHub 인라인 코멘트

**현재 상태:** PR 하단에 전체 요약 코멘트 1개만 게시
**목표:** 스킬 에이전트가 발견한 이슈를 코드 특정 라인에 직접 인라인 코멘트로 게시

**구현 포인트:**
- GitHub `POST /repos/{owner}/{repo}/pulls/{pull_number}/reviews` API 활용
- 이슈별 `path` (파일경로) + `line` (라인 번호) 추출 필요
- 스킬 에이전트 출력에 `line` 필드 추가
- 기존 요약 코멘트는 유지 (body 필드)

**기대 효과:** 사용자 체감 리뷰 품질 최대 향상. 어떤 라인이 문제인지 즉시 확인 가능.

## 🔴 P2 — 토큰 / 비용 추적

**현재 상태:** LLM 호출 비용 추적 없음
**목표:** 리뷰별 토큰 사용량과 예상 비용을 DB에 저장하고 대시보드에 표시

**구현 포인트:**
- `reviews` 테이블에 `input_tokens`, `output_tokens`, `estimated_cost_usd` 컬럼 추가
- LangChain response의 `usage_metadata` 활용
- 스킬별 토큰 사용량 집계 (어떤 스킬이 가장 비싼지)
- 대시보드 Stats 카드에 월별 비용 표시

**기대 효과:** 스킬 수 × 파일 수 병렬 구조에서 과금 현황 직접 파악 가능.

## 🟡 P3 — 스킬별 모델 선택

**현재 상태:** 모든 스킬이 동일 모델 사용
**목표:** 스킬별로 다른 LLM 모델 지정 가능

**구현 포인트:**
- `skills` 테이블에 `model_override` 컬럼 추가 (nullable)
- `_run_skill_agent()`에서 skill의 model_override 참조
- UI에 모델 선택 드롭다운 추가

**예시:**
```
보안 점검    → claude-opus-4-6  (비싸지만 정확)
코드 스타일  → claude-haiku-4-5 (빠르고 저렴)
```

## 🟡 P4 — 스킬 성능 분석 대시보드

**현재 상태:** `is_addressed` 데이터가 쌓이고 있지만 활용 없음
**목표:** 스킬별 이슈 발견 수 / 실제 수정률 시각화

**구현 포인트:**
- `review_comments` 테이블의 skill 태그 + `is_addressed` 집계
- 스킬별 precision 지표 (addressed / total)
- 저노이즈 스킬 vs 고가치 스킬 구분 가능

---

## 🟢 P5 — RAG 코드베이스 컨텍스트

**현재 상태:** 서브 에이전트는 변경된 diff만 참조
**목표:** 변경된 함수의 호출부, 관련 인터페이스 등을 벡터 검색으로 주입

**구현 포인트:**
- 저장소 코드베이스 임베딩 (pgvector 또는 외부 벡터 DB)
- PR 변경 내용 기반 관련 코드 검색
- 스킬 에이전트 프롬프트에 관련 코드 컨텍스트 추가

---

## 🟢 P6 — 스킬 템플릿 마켓플레이스

**현재 상태:** 스킬을 처음부터 직접 작성해야 함
**목표:** 검증된 스킬 템플릿을 원클릭으로 저장소에 추가

**구현 포인트:**
- 템플릿 테이블 또는 정적 JSON 파일로 관리
- 언어/프레임워크별 템플릿 분류 (Python, FastAPI, React, Django 등)
- UI에 "템플릿에서 추가" 플로우

---

## 🟢 P7 — 리뷰 큐 + 우선순위 처리

**현재 상태:** 웹훅 수신 즉시 동기 실행
**목표:** 비동기 큐 기반 처리, HIGH 위험도 PR 우선 처리

**구현 포인트:**
- ARQ(Redis 기반) 또는 Celery 도입
- 위험도 기반 우선순위 큐
- 처리 상태를 PR 페이지에 실시간 표시 (WebSocket 또는 polling)
